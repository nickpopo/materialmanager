# Architecture Overview

## Основные компоненты

Проект разделен на несколько модулей, каждый из которых отвечает за определенные функции:

### 1. **GUI (src/gui.py)**

Этот модуль реализует графический интерфейс пользователя на основе Tkinter. Основные компоненты интерфейса включают:

- **Login Interface**: Экран входа и регистрации.
- **Main Interface**: Основной интерфейс с управлением материалами и отчетами.

### 2. **Database (src/database.py)**

Этот модуль отвечает за взаимодействие с базой данных SQLite. Включает функции для создания подключения к базе данных и управления таблицами.

- **create_connection**: Создает подключение к базе данных.
- **create_tables**: Создает таблицы `users` и `materials`.

### 3. **Report (src/report.py)**

Этот модуль отвечает за генерацию отчетов и их экспорт в формат Excel.

- **export_to_excel**: Экспортирует данные о материалах из базы данных в Excel.

## Структура базы данных

Проект использует SQLite для хранения данных. База данных содержит две таблицы:

1. **users**
   - `id` (INTEGER, PRIMARY KEY)
   - `username` (TEXT, UNIQUE)
   - `password` (TEXT)

2. **materials**
   - `id` (INTEGER, PRIMARY KEY)
   - `name` (TEXT)
   - `barcode` (TEXT)
   - `quantity` (INTEGER)
   - `unit` (TEXT)

## Потоки данных

1. Пользователь взаимодействует с интерфейсом.
2. Введенные данные обрабатываются модулем GUI.
3. Модуль GUI взаимодействует с базой данных через модуль `database.py`.
4. Результаты операций отображаются на интерфейсе или сохраняются в виде отчета через модуль `report.py`.

## Зависимости

Проект использует следующие библиотеки:

- **tkinter**: Для создания графического интерфейса.
- **sqlite3**: Для взаимодействия с базой данных.
- **openpyxl**: Для экспорта данных в Excel.

Зависимости перечислены в файле `requirements.txt`.
